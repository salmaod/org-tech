3. Chargement des données et aperçu général¶

Description des colonnes¶

  • Unnamed: 0 : Représente la date des ventes (cette colonne sera convertie au format datetime pour une meilleure manipulation).
  • marketing_score : Indique l'intensité et l'efficacité des actions marketing mises en œuvre (campagnes, promotions).
  • competition_index : Mesure le niveau de pression concurrentielle dans le marché.
  • customer_satisfaction : Score représentant la satisfaction des clients (mesuré sur une échelle prédéfinie).
  • purchasing_power_index : Indice du pouvoir d'achat dans la région où la vente a eu lieu.
  • weather_condition : État des conditions météorologiques pour un jour donné (exemple : "Good", "Bad", "Moderate").
  • tech_event : Variable binaire ou catégorique indiquant s'il s'agit d'un jour avec un événement technologique significatif.
  • 5g_phase : Phase de déploiement de la technologie 5G (par exemple, "Pre-5G", "Mature-5G").
  • store_traffic : Score représentant l'affluence des clients dans les magasins.
  • public_transport : Qualité ou état des transports publics locaux (exemple : "Hub", "Good", "Limited").
  • city : Ville où la transaction ou l'événement de vente a été enregistré.
  • jPhone_Pro_revenue : Revenus générés par le modèle jPhone Pro sur une période donnée.
  • Kaggle_Pixel_5_revenue : Revenus générés par le modèle Kaggle Pixel 5 sur une période donnée.
  • Planet_SX_revenue : Revenus générés par le modèle Planet SX sur une période donnée.

4. Statistiques descriptives¶

Explications des colonnes des statistiques descriptives :¶

Pour les variables numériques :¶

  • Colonne : Nom de la colonne analysée.
  • count : Nombre total de valeurs non manquantes (observations) dans la colonne.
  • mean : Moyenne (valeur moyenne) des données.
  • std : Écart-type, qui mesure la dispersion des valeurs par rapport à la moyenne.
  • min : Valeur minimale dans la colonne.
  • 25% (1er Quartile) : La valeur en dessous de laquelle se trouvent 25 % des données.
  • 50% (Médiane) : La valeur médiane (valeur centrale) des données.
  • 75% (3e Quartile) : La valeur en dessous de laquelle se trouvent 75 % des données.
  • max : Valeur maximale dans la colonne.

Pour les variables catégoriques :¶

  • Colonne : Nom de la colonne analysée.
  • count : Nombre total de valeurs non manquantes (observations) dans la colonne.
  • unique : Nombre de valeurs uniques présentes dans la colonne.
  • top : Valeur la plus fréquente (mode) dans la colonne.
  • freq : Fréquence d'apparition de la valeur la plus fréquente.

from src.data_processing import clean_data

# Chargement des données
file_path = "data/raw/telecom_sales_data.csv"
data = pd.read_csv(file_path)

# Vérification des valeurs manquantes AVANT nettoyage
missing_data_raw = data.isnull().sum().reset_index()
missing_data_raw.columns = ['Colonne', 'Valeurs manquantes (Avant)']
missing_data_raw['Pourcentage (Avant)'] = (missing_data_raw['Valeurs manquantes (Avant)'] / len(data) * 100).round(2)

# Nettoyage des données
data_cleaned = clean_data(data)

# Vérification des valeurs manquantes APRES nettoyage
missing_data_after = data_cleaned.isnull().sum().reset_index()
missing_data_after.columns = ['Colonne', 'Valeurs manquantes (Après)']
missing_data_after['Pourcentage (Après)'] = (missing_data_after['Valeurs manquantes (Après)'] / len(data_cleaned) * 100).round(2)

# Combiner les tableaux avant/après
combined_missing_data = pd.merge(missing_data_raw, missing_data_after, on="Colonne")
print("\nValeurs manquantes par colonne (avant et après nettoyage) :\n")
print(tabulate(combined_missing_data, headers="keys", tablefmt="fancy_grid"))

Observations¶

La matrice de corrélation permet de visualiser les relations linéaires entre les variables numériques. Les corrélations élevées sont particulièrement visibles entre :

9. Préparation des insights pour la modélisation¶

1. Nettoyage des données :¶

  • Problèmes détectés :

    • Données manquantes : Notamment dans les variables liées aux événements technologiques (tech_event) et certaines mesures clés comme les revenus.
    • Valeurs aberrantes : Observées principalement dans les revenus du modèle premium (jPhone Pro), nécessitant des ajustements.
    • Distributions non homogènes : Variabilité significative entre les modèles de smartphones et les villes.

  • Actions entreprises :

    • Traitement des valeurs manquantes par suppression ou imputation selon le contexte.
    • Identification et ajustement des outliers pour garantir une meilleure stabilité des analyses.
    • Normalisation des variables critiques pour une cohérence dans les étapes de modélisation.

2. Insights principaux :¶

  • Revenus par modèle :

    • jPhone Pro domine en termes de revenus, avec une forte dispersion indiquant une variabilité dans les performances.
    • Kaggle Pixel 5 affiche des revenus moyens, mais plus homogènes, ce qui en fait un modèle stable.
    • Planet SX a les revenus les plus faibles, mais une performance constante.

  • Variabilité géographique :

    • Les villes comme Paris et Lyon contribuent significativement aux revenus totaux.
    • Les différences de pouvoir d'achat et de trafic en magasin influencent directement les revenus par région.

3. Relations entre variables :¶

  • Corrélations clés :

    • Le Marketing Score présente une corrélation positive forte avec les revenus pour tous les modèles, en particulier jPhone Pro.
    • Le purchasing_power_index et le store_traffic montrent un impact direct sur les ventes dans toutes les villes.
    • Les conditions météorologiques favorables (Good) amplifient les revenus, surtout lors de pics.

  • Interactions temporelles :

    • Des tendances saisonnières claires émergent, suggérant l'impact d'événements ou promotions spécifiques.
    • Les périodes de pics révèlent des opportunités de campagnes marketing ciblées.

4. Étapes préparatoires pour la modélisation :¶

  • Sélection des variables les plus influentes pour limiter la complexité des modèles tout en optimisant leurs performances.
  • Construction de nouvelles variables dérivées (par exemple, indicateurs saisonniers ou régionaux).
  • Garantir l'équilibre des données à travers les modèles pour éviter des biais dans les prédictions.

Conclusion : Cette analyse approfondie des données brutes fournit une base robuste et interprétable pour les prochaines étapes de modélisation. Les insights extraits orienteront les choix méthodologiques et stratégiques pour maximiser la pertinence des prédictions.
